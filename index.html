Welcome to CODDI.

This is template for your Drupal-based project, that covers most development operations, servers configurations and continous deployments. That is why we call it Complete Drupal Development Infrastructure or Continous Drupal Delivery Infrastructure.

CODDI use drush, phing, vagrant, chef and jenkins to automate whole development cycle for Drupal projects. It makes developers, testers and sysadmins to speak on the "same language" to reach the goal of the project.

I'm a developer. How would it help me?

CODDI will standardize and automate most of common Drupal operations. It provide:

  • developer local VM equal to production server
  • easy way to work with community and third party code
  • easy to apply patches
  • way to enable automate modules
  • way to sync state of DB with code in one command
  • way to up complex Drupal site from scratch
  • way run the same changes during deploy in auto mode
  • You deploys will became stress less.

I'm system admin. How would it help me?

  • You will have scratch for Drupal servers infrastructure
  • You will have configuration of jenkins server with scratch pipe line that makes automated deployments.
  • You will write chef code once, and every developers in team will get changes.

I'm QA. How would it help me?

  • You will have developer local VM equal to production server
  • You will have one command to update your local Drupal project to latest version created by developers.
  • You will have way to test changes on local VM and block it's merge to common branch. Such workflow will make common branch stable and quality of project will up.
  • In nearly future we going to make TDD for Drupal real with CODDI.

I'm manager. How would it help me?

  • Read all over, to find that CODDI will reduce chaos in your project.
  • CODDI will speed up your team by making routing work with CODDI
